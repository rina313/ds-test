### 📦 IconProps 확장 사용 예제

`Icon.types.ts`의 `IconProps` 타입을 기반으로, 특정 컴포넌트에 아이콘 관련 props를 확장할 수 있습니다.
<br/>예를 들어 `Component`에 `IconProps`타입을 가진 `trailingIcon`이라는 prop을 사용한다고 가정할 때:

---
<br/>
#### 🧱 타입 확장 방식

```ts
import { ExtendIconProps } from './Icon.types';

type ComponentExtendIconProps = ExtendIconProps<ComponentProps, 'trailingIcon'>;
```

위 코드는 다음과 같은 props를 생성합니다:

```ts
{
  trailingIcon?: IconProps;
  trailingIconName?: IconName;
  trailingIconVariants?: IconVariant;
  trailingIconSrc?: string;
}
```

---

#### ⚙️ Storybook Meta 구성

`icon.meta.ts`에 정의된 헬퍼로 meta를 생성할 수 있습니다:

```ts
import { getIconMeta } from './icon.meta';

const trailingIconMeta = getIconMeta('trailingIcon');
```

`trailingIconMeta`는 다음과 같은 구조를 가집니다:

```ts
{
  args: {
    trailingIcon: undefined,
    trailingIconName: 'expand',
    trailingIconVariants: 'fa-solid',
    trailingIconSrc: undefined,
  },
  argTypes: {
    trailingIcon: { control: false },
    trailingIconName: { control: 'select', ... },
    trailingIconVariants: { control: 'select', ... },
    trailingIconSrc: { control: 'text' },
  }
}
```

---

#### 🧪 Story 예시

```tsx
export const Primary: Story = {
  args: {
    // ...
  },
  render: (args) => {
    const {
      trailingIconName,
      trailingIconVariants,
      trailingIconSrc,
      ...rest
    } = args;

    return (
      <Component
        {...rest}
        trailingIcon={{
          iconName: trailingIconName,
          variants: trailingIconVariants,
          src: trailingIconSrc,
        }}
      />
    );
  },
};
```

---

#### ✅ 요약

- `ExtendIconProps<T, 'key'>`를 활용해 재사용 가능한 icon 관련 prop을 정의할 수 있습니다.
- `getIconMeta('key')`를 통해 `args`/`argTypes`를 자동 생성할 수 있어 Storybook 구성도 간결해집니다.
- 스토리에서는 `render` 함수에서 `props`를 병합하여 원하는 형태로 전달하면 됩니다.
- 단, `IconProps` 타입을 사용하는 모든 `props`에 새로운 `meta`데이터와 `Type`을 적용해야 합니다.
